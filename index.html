      Graph Theory Concepts

      ⟶

      Directed Graph (Digraph)

      A directed graph G = (V, E) consists of a set of vertices V and a set of ordered pairs called directed edges E. Each edge (u, v) points from u to v and not necessarily the reverse — unlike undirected graphs where edges are bidirectional.

      G = ({A,B,C}, {(A→B),(B→C),(A→C)})
      ⚖

      Weighted Edges

      A weighted graph assigns a numeric cost or weight w(u,v) to each edge. Weights can represent distance, time, similarity, or strength of a relationship. In knowledge graphs, a higher weight often means a stronger conceptual link.

      w(AI → ML) = 1 (direct parent)
      w(AI → DL) = 3 (indirect path)
      🔍

      Graph Traversal — BFS & DFS

      BFS (Breadth-First Search) explores all neighbours at distance 1, then 2, etc., making it ideal for finding the shortest unweighted path. DFS dives deep along one branch before backtracking, suitable for cycle detection and topological sort.

      BFS queue: [A] → [B,C] → [D,E]
      DFS stack: A → B → D → backtrack
      📐

      Dijkstra's Algorithm

      Finds the minimum-weight path between a source and all other nodes. It uses a priority queue: always relax the node with the smallest tentative distance. Time complexity: O((V + E) log V) with a binary heap.

      dist[src]=0; dist[others]=∞
      while queue: relax neighbours
      🔗

      In-degree & Out-degree

      The in-degree of a node is the number of edges pointing into it; the out-degree is the number pointing out. Nodes with high in-degree (hubs) are conceptually important — they are depended upon by many others.

      in-deg(ML) = 1 (from AI)
      out-deg(ML) = 2 (to DL, NLP)
      🔄

      Cycles & DAGs

      A cycle exists when you can follow directed edges and return to the starting node. A graph with no cycles is called a DAG (Directed Acyclic Graph). Knowledge graphs are often DAGs — concepts build on each other without circular dependency.

      Cycle: A→B→C→A
      DAG: A→B→C (no return path)

      Dijkstra's Step-by-Step Walkthrough

      1. Initialise — set dist[source] = 0, all others = ∞. Insert all nodes into a min-priority queue keyed by distance.
      2. Extract minimum — pop the node u with the smallest distance.
      3. Relax neighbours — for each neighbour v of u, if dist[u] + w(u,v) < dist[v], update dist[v] and record predecessor[v] = u.
      4. Repeat until the queue is empty or the destination is popped.
      5. Reconstruct path — follow predecessor[] backwards from destination to source.

      Real-World Applications

      🧠

      Google Knowledge Graph

      Powers the info panels on the right side of Google Search. When you search "Einstein", the knowledge graph surfaces his birth date, works, and relationships without you having to visit a page.

      Search & Discovery
      💊

      Drug Interaction Networks

      Pharmaceutical companies model drugs, diseases, proteins, and genes as nodes. Edges encode interactions, enabling AI to predict adverse effects or repurpose existing drugs for new diseases.

      Healthcare
      🛒

      Recommendation Systems

      Amazon and Netflix use graph-based collaborative filtering: users, products, and genres are nodes; purchases or views are weighted edges. Shortest-path similarity drives "customers also bought" suggestions.

      E-commerce / Media
      🔐

      Fraud Detection

      Banks map accounts, transactions, and devices as nodes. Unusual clusters or short cycles (A→B→C→A in money flows) signal laundering rings, enabling real-time fraud alerts.

      Finance
      🗺

      Navigation & Route Planning

      Google Maps is a giant weighted directed graph. Dijkstra (and its variants A* and Bidirectional Dijkstra) compute the fastest route by minimising total edge weight (time or distance).

      Logistics
      📚

      Curriculum Design

      MOOCs like Coursera model prerequisite chains as DAGs. A topological sort of the graph gives the optimal learning order — just like the AI → ML → DL sample in this app.

      Education

      How This App Works — Code Architecture

      01

      Data Model

      Two arrays — nodes[] and edges[] — are the source of truth. Each node carries an id, label, and category. Each edge carries source, target, type, and weight.

      02

      D3 Force Layout

      D3's forceSimulation runs a physics engine: charge repulsion keeps nodes apart, link forces pull connected nodes together, and centering keeps the graph visible.

      03

      SVG Rendering

      Nodes are <circle> elements; edges are <line> elements with <marker> arrowheads. A <g> zoom/pan container wraps everything.

      04

      Dijkstra

      Implemented from scratch with a simple priority queue. The adjacency list is built fresh on each run from the edges[] array, so it always reflects the current graph state.

      05

      Quiz Engine

      A question is generated by picking a random edge and asking what relation connects the two nodes. Three distractors are sampled from other edge types to form the MCQ.

      How to Run the Project

      1. Download all three files — index.html, style.css, script.js — into the same folder.
      2. Open index.html in any modern browser (Chrome, Firefox, Edge, Safari). No server needed — D3.js loads from a CDN.
      3. Click "Load Default Graph" in the sidebar to instantly populate the AI → ML → DL graph.
      4. Drag nodes, scroll to zoom, click a node to see its connections, or run Dijkstra between any two nodes.
      5. Switch to the Quiz tab to test your knowledge of the loaded graph.
